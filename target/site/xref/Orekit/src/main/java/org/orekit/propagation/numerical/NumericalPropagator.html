49  /** This class propagates {@link org.orekit.orbits.Orbit orbits} using
50   * numerical integration.
51   * <p>Numerical propagation is much more accurate than analytical propagation
52   * like for example {@link org.orekit.propagation.analytical.KeplerianPropagator
53   * keplerian} or {@link org.orekit.propagation.analytical.EcksteinHechlerPropagator
54   * Eckstein-Hechler}, but requires a few more steps to set up to be used properly.
55   * Whereas analytical propagators are configured only thanks to their various
56   * constructors and can be used immediately after construction, numerical propagators
57   * configuration involve setting several parameters between construction time
58   * and propagation time.</p>
59   * <p>The configuration parameters that can be set are:</p>
60   * <ul>
61   *   <li>the initial spacecraft state ({@link #setInitialState(SpacecraftState)})</li>
62   *   <li>the central attraction coefficient ({@link #setMu(double)})</li>
63   *   <li>the various force models ({@link #addForceModel(ForceModel)},
64   *   {@link #removeForceModels()})</li>
65   *   <li>the {@link OrbitType type} of orbital parameters to be used for propagation
66   *   ({@link #setOrbitType(OrbitType)}),
67   *   <li>the {@link PositionAngle type} of position angle to be used in orbital parameters
68   *   to be used for propagation where it is relevant ({@link
69   *   #setPositionAngleType(PositionAngle)}),
70   *   <li>whether {@link org.orekit.propagation.integration.AdditionalEquations additional equations}
71   *   (for example {@link PartialDerivativesEquations Jacobians}) should be propagated along with orbital state
72   *   ({@link #addAdditionalEquations(org.orekit.propagation.integration.AdditionalEquations)}),
73   *   <li>the discrete events that should be triggered during propagation
74   *   ({@link #addEventDetector(EventDetector)},
75   *   {@link #clearEventsDetectors()})</li>
76   *   <li>the binding logic with the rest of the application ({@link #setSlaveMode()},
77   *   {@link #setMasterMode(double, org.orekit.propagation.sampling.OrekitFixedStepHandler)},
78   *   {@link #setMasterMode(org.orekit.propagation.sampling.OrekitStepHandler)},
79   *   {@link #setEphemerisMode()}, {@link #getGeneratedEphemeris()})</li>
80   * </ul>
81   * <p>From these configuration parameters, only the initial state is mandatory. The default
82   * propagation settings are in {@link OrbitType#EQUINOCTIAL equinoctial} parameters with
83   * {@link PositionAngle#TRUE true} longitude argument. If the central attraction coefficient
84   * is not explicitly specified, the one used to define the initial orbit will be used.
85   * However, specifying only the initial state and perhaps the central attraction coefficient
86   * would mean the propagator would use only keplerian forces. In this case, the simpler {@link
87   * org.orekit.propagation.analytical.KeplerianPropagator KeplerianPropagator} class would
88   * perhaps be more effective.</p>
89   * <p>The underlying numerical integrator set up in the constructor may also have its own
90   * configuration parameters. Typical configuration parameters for adaptive stepsize integrators
91   * are the min, max and perhaps start step size as well as the absolute and/or relative errors
92   * thresholds.</p>
93   * <p>The state that is seen by the integrator is a simple seven elements double array.
94   * The six first elements are either:
95   * <ul>
96   *   <li>the {@link org.orekit.orbits.EquinoctialOrbit equinoctial orbit parameters} (a, e<sub>x</sub>,
97   *   e<sub>y</sub>, h<sub>x</sub>, h<sub>y</sub>, λ<sub>M</sub> or λ<sub>E</sub>
98   *   or λ<sub>v</sub>) in meters and radians,</li>
99   *   <li>the {@link org.orekit.orbits.KeplerianOrbit Keplerian orbit parameters} (a, e, i, ω, Ω,
100  *   M or E or v) in meters and radians,</li>
101  *   <li>the {@link org.orekit.orbits.CircularOrbit circular orbit parameters} (a, e<sub>x</sub>, e<sub>y</sub>, i,
102  *   Ω, α<sub>M</sub> or α<sub>E</sub> or α<sub>v</sub>) in meters
103  *   and radians,</li>
104  *   <li>the {@link org.orekit.orbits.CartesianOrbit Cartesian orbit parameters} (x, y, z, v<sub>x</sub>,
105  *   v<sub>y</sub>, v<sub>z</sub>) in meters and meters per seconds.
106  * </ul>
107  * The last element is the mass in kilograms.
108  * </p>
109  * <p>The following code snippet shows a typical setting for Low Earth Orbit propagation in
110  * equinoctial parameters and true longitude argument:</p>
111  * <pre>
112  * final double dP       = 0.001;
113  * final double minStep  = 0.001;
114  * final double maxStep  = 500;
115  * final double initStep = 60;
116  * final double[][] tolerance = NumericalPropagator.tolerances(dP, orbit, OrbitType.EQUINOCTIAL);
117  * AdaptiveStepsizeIntegrator integrator = new DormandPrince853Integrator(minStep, maxStep, tolerance[0], tolerance[1]);
118  * integrator.setInitialStepSize(initStep);
119  * propagator = new NumericalPropagator(integrator);
120  * </pre>
121  * <p>The same propagator can be reused for several orbit extrapolations, by resetting
122  * the initial state without modifying the other configuration parameters. However, the
123  * same instance cannot be used simultaneously by different threads, the class is <em>not</em>
124  * thread-safe.</p>
125 
126  * @see SpacecraftState
127  * @see ForceModel
128  * @see org.orekit.propagation.sampling.OrekitStepHandler
129  * @see org.orekit.propagation.sampling.OrekitFixedStepHandler
130  * @see org.orekit.propagation.integration.IntegratedEphemeris
131  * @see TimeDerivativesEquations
132  *
133  * @author Mathieu Rom&eacute;ro
134  * @author Luc Maisonobe
135  * @author Guylaine Prat
136  * @author Fabien Maussion
137  * @author V&eacute;ronique Pommier-Maurussane
138  */

147     /** Create a new instance of NumericalPropagator, based on orbit definition mu.
148      * After creation, the instance is empty, i.e. the attitude provider is set to an
149      * unspecified default law and there are no perturbing forces at all.
150      * This means that if {@link #addForceModel addForceModel} is not
151      * called after creation, the integrated orbit will follow a keplerian
152      * evolution only. The defaults are {@link OrbitType#EQUINOCTIAL}
153      * for {@link #setOrbitType(OrbitType) propagation
154      * orbit type} and {@link PositionAngle#TRUE} for {@link
155      * #setPositionAngleType(PositionAngle) position angle type}.
156      * @param integrator numerical integrator to use for propagation.
157      */
158     public NumericalPropagator(final AbstractIntegrator integrator) {
159         super(integrator, true);
160         forceModels = new ArrayList<ForceModel>();
161         initMapper();
162         setAttitudeProvider(DEFAULT_LAW);
163         setMu(Double.NaN);
164         setSlaveMode();
165         setOrbitType(OrbitType.EQUINOCTIAL);
166         setPositionAngleType(PositionAngle.TRUE);
167     }

307         /** {@inheritDoc} */
308         public SpacecraftState mapArrayToState(final double t, final double[] y, final boolean meanOnly)
309             throws OrekitException {
310             // the parameter meanOnly is ignored for the Numerical Propagator
311 
312             final double mass = y[6];
313             if (mass <= 0.0) {
314                 throw new PropagationException(OrekitMessages.SPACECRAFT_MASS_BECOMES_NEGATIVE, mass);
315             }
316 
317             final AbsoluteDate date = mapDoubleToDate(t);
318             final Orbit orbit       = getOrbitType().mapArrayToOrbit(y, getPositionAngleType(), date, getMu(), getFrame());
319             final Attitude attitude = getAttitudeProvider().getAttitude(orbit, date, getFrame());
320 
321             return new SpacecraftState(orbit, attitude, mass);
322 
323         }

326         public void mapStateToArray(final SpacecraftState state, final double[] y) {
327             getOrbitType().mapOrbitToArray(state.getOrbit(), getPositionAngleType(), y);
328             y[6] = state.getMass();
329         }

398     /** Internal class for osculating parameters integration. */
399     private class Main implements MainStateEquations, TimeDerivativesEquations {
400 
401         /** Derivatives array. */
402         private final double[] yDot;
403 
404         /** Current orbit. */
405         private Orbit orbit;
406 
407         /** Jacobian of the orbital parameters with respect to the cartesian parameters. */
408         private double[][] jacobian;
409 
410         /** Simple constructor.
411          * @param integrator numerical integrator to use for propagation.
412          */
413         public Main(final AbstractIntegrator integrator) {
414 
415             this.yDot     = new double[7];
416             this.jacobian = new double[6][6];
417 
418             for (final ForceModel forceModel : forceModels) {
419                 final EventDetector[] modelDetectors = forceModel.getEventsDetectors();
420                 if (modelDetectors != null) {
421                     for (final EventDetector detector : modelDetectors) {
422                         setUpEventDetector(integrator, detector);
423                     }
424                 }
425             }
426 
427         }
428 
429         /** {@inheritDoc} */
430         public double[] computeDerivatives(final SpacecraftState state) throws OrekitException {
431 
432             orbit = state.getOrbit();
433             Arrays.fill(yDot, 0.0);
434             orbit.getJacobianWrtCartesian(getPositionAngleType(), jacobian);
435 
436             // compute the contributions of all perturbing forces
437             for (final ForceModel forceModel : forceModels) {
438                 forceModel.addContribution(state, this);
439             }
440 
441             // finalize derivatives by adding the Kepler contribution
442             newtonianAttraction.addContribution(state, this);
443 
444             return yDot.clone();
445 
446         }
447 
448         /** {@inheritDoc} */
449         public void addKeplerContribution(final double mu) {
450             orbit.addKeplerContribution(getPositionAngleType(), mu, yDot);
451         }
452 
453         /** {@inheritDoc} */
454         public void addXYZAcceleration(final double x, final double y, final double z) {
455             for (int i = 0; i < 6; ++i) {
456                 final double[] jRow = jacobian[i];
457                 yDot[i] += jRow[3] * x + jRow[4] * y + jRow[5] * z;
458             }
459         }
460 
461         /** {@inheritDoc} */
462         public void addAcceleration(final Vector3D gamma, final Frame frame)
463             throws OrekitException {
464             final Transform t = frame.getTransformTo(orbit.getFrame(), orbit.getDate());
465             final Vector3D gammInRefFrame = t.transformVector(gamma);
466             addXYZAcceleration(gammInRefFrame.getX(), gammInRefFrame.getY(), gammInRefFrame.getZ());
467         }
468 
469         /** {@inheritDoc} */
470         public void addMassDerivative(final double q) {
471             if (q > 0) {
472                 throw new OrekitIllegalArgumentException(OrekitMessages.POSITIVE_FLOW_RATE, q);
473             }
474             yDot[6] += q;
475         }
476 
477     }

479     /** Estimate tolerance vectors for integrators.
480      * <p>
481      * The errors are estimated from partial derivatives properties of orbits,
482      * starting from a scalar position error specified by the user.
483      * Considering the energy conservation equation V = sqrt(mu (2/r - 1/a)),
484      * we get at constant energy (i.e. on a Keplerian trajectory):
485      * <pre>
486      * V² r |dV| = mu |dr|
487      * </pre>
488      * So we deduce a scalar velocity error consistent with the position error.
489      * From here, we apply orbits Jacobians matrices to get consistent errors
490      * on orbital parameters.
491      * </p>
492      * <p>
493      * The tolerances are only <em>orders of magnitude</em>, and integrator tolerances
494      * are only local estimates, not global ones. So some care must be taken when using
495      * these tolerances. Setting 1mm as a position error does NOT mean the tolerances
496      * will guarantee a 1mm error position after several orbits integration.
497      * </p>
498      * @param dP user specified position error
499      * @param orbit reference orbit
500      * @param type propagation type for the meaning of the tolerance vectors elements
501      * (it may be different from {@code orbit.getType()})
502      * @return a two rows array, row 0 being the absolute tolerance error and row 1
503      * being the relative tolerance error
504      * @exception PropagationException if Jacobian is singular
505      */
506     public static double[][] tolerances(final double dP, final Orbit orbit, final OrbitType type)
507         throws PropagationException {
508 
509         // estimate the scalar velocity error
510         final PVCoordinates pv = orbit.getPVCoordinates();
511         final double r2 = pv.getPosition().getNormSq();
512         final double v  = pv.getVelocity().getNorm();
513         final double dV = orbit.getMu() * dP / (v * r2);
514 
515         final double[] absTol = new double[7];
516         final double[] relTol = new double[7];
517 
518         // we set the mass tolerance arbitrarily to 1.0e-6 kg, as mass evolves linearly
519         // with trust, this often has no influence at all on propagation
520         absTol[6] = 1.0e-6;
521 
522         if (type == OrbitType.CARTESIAN) {
523             absTol[0] = dP;
524             absTol[1] = dP;
525             absTol[2] = dP;
526             absTol[3] = dV;
527             absTol[4] = dV;
528             absTol[5] = dV;
529         } else {
530 
531             // convert the orbit to the desired type
532             final double[][] jacobian = new double[6][6];
533             final Orbit converted = type.convertType(orbit);
534             converted.getJacobianWrtCartesian(PositionAngle.TRUE, jacobian);
535 
536             for (int i = 0; i < 6; ++i) {
537                 final double[] row = jacobian[i];
538                 absTol[i] = FastMath.abs(row[0]) * dP +
539                             FastMath.abs(row[1]) * dP +
540                             FastMath.abs(row[2]) * dP +
541                             FastMath.abs(row[3]) * dV +
542                             FastMath.abs(row[4]) * dV +
543                             FastMath.abs(row[5]) * dV;
544                 if (Double.isNaN(absTol[i])) {
545                     throw new PropagationException(OrekitMessages.SINGULAR_JACOBIAN_FOR_ORBIT_TYPE, type);
546                 }
547             }
548 
549         }
550 
551         Arrays.fill(relTol, dP / FastMath.sqrt(r2));
552 
553         return new double[][] {
554             absTol, relTol
555         };
556 
557     }
